SYSTEM ARCHITECTURE

三个层级，避免“每个零件各自为政”

所有部件都有身份、算力与安全状态，但决策能力按实时性分层。

L1

零件层

关节控制、触觉、温度、锁止检测、毫秒级急停。

  • 1–5 ms 控制周期
  • 本地故障隔离
  • 模块身份证
→
L2

载体层

机器人全身协调、无人机飞控、避碰、动力学模型。

  • 身体/飞行稳定
  • 质量与惯量更新
  • 任务执行
→
L3

系统层

语言理解、长期规划、模块调度与多智能体协作。

  • 任务分配
  • 技能库
  • 数字孪生

DELIVERY ROADMAP

从概念演示到工程验证

先验证最有商业价值、风险最可控的模块迁移，再进入无人机与全身人形。

01

视觉定义

确定机器人外观语言、关节形式、接口剖面、无人机比例和镜头脚本。

交付：风格板 + 分镜 + 关键帧
02

交互原型

将每个难点绑定时间码、画面坐标、说明和建议实现。

交付：网页交互 + 演示模式
03

3D 动画

用 Blender / Three.js 替换示意 SVG，加入材质、机械锁与真实关节运动。

交付：60–90 秒概念片
04

工程数字孪生

接入真实 CAD、URDF、载荷与控制数据，使视觉演示逐步成为验证工具。

交付：参数化仿真界面
