# Codex 主指令：模块化智能机器人可视化项目

你正在维护一个纯静态前端项目，用于展示“机器人头、手、手臂、腿脚、工具和无人机之间通过统一接口迁移”的产品构想。

## 现有产品定义

- 磁力负责捕获与自动找正。
- 机械锁负责承载与断电保持。
- 接口同时传递电源、实时控制数据、高速数据、身份和独立安全信号。
- 每个部件是智能模块，但采用三级智能：零件层、载体层、系统层。
- 产品按四阶段推进：
  1. 手与工具快换
  2. 整条机械臂跨载体迁移
  3. 无人机搭载专用轻量小臂
  4. 完整模块化人形

## 当前技术约束

- 保持纯静态部署，不引入后端。
- 优先使用原生 HTML、CSS、JavaScript 和 SVG。
- 所有功能必须支持 320px 到桌面宽度。
- 不使用需要 API Key 的资源。
- 不删除已有阶段、难点、验收标准和架构内容。
- 每次改动后检查浏览器控制台错误。

## 近期迭代顺序

### 任务 1：升级机器人视觉细节

目标：让机器人从“概念图标”升级为“具有工程可信度的机械设计示意”。

要求：
- 头部加入颈部俯仰/旋转关节、双目相机、深度传感器、散热与连接环。
- 手臂加入肩部 3 自由度表达、肘关节减速器、腕部 3 自由度表达、走线和接口锁止环。
- 手部加入掌部驱动、拇指基座和每根手指至少两个可见关节。
- 腿部加入髋、膝、踝和足底受力区域。
- 关节结构在四个阶段中保持同一种设计语言。
- 不追求写实材质，优先保证结构清楚、动画稳定和可点击高亮。

验收：
- 各主要关节在 1366×768 和 390×844 两种视口均清晰可辨。
- 难点圈选位置准确。
- 动画播放时关节不会与主体明显脱节。

### 任务 2：无人机渐进式进入

目标：不是突然出现一台无人机，而是展示其逐步成为兼容载体。

要求：
- 阶段 1：只出现“未来载体”轮廓或接口占位，不抢主叙事。
- 阶段 2：在实验区背景中出现无人机底座与接口验证状态。
- 阶段 3：展示无人机框架展开、轻量机械臂接入、视觉锁定、接触抓取。
- 阶段 4：无人机作为独立智能载体，与人形机器人共享任务网络。
- 为无人机增加载荷、续航、重心和反作用力四类可点击难点。

### 任务 3：接口剖面模式

目标：用户点击任意连接环时，打开一个侧边剖面视图。

剖面必须展示：
- 永磁/电永磁捕获层
- 锥面或定位销
- 双机械锁
- 电源预充电触点
- 实时总线触点
- 高速数据触点
- 独立安全线
- 锁止和温度传感器

交互：
- 点击连接环打开。
- 鼠标或触屏选择各层并查看说明。
- Esc、关闭按钮和点击遮罩均可关闭。
- 移动端为底部抽屉，桌面端为右侧抽屉。

### 任务 4：3D 迁移准备

在不立即引入 Three.js 的前提下，重构数据结构：
- 阶段、场景对象、关节、接口、难点和时间线关键帧分离。
- 每个热点使用稳定 id，不依赖数组序号。
- 为未来 Three.js 场景保留 position、rotation、scale、parentId 字段。
- 确保现有 SVG 渲染不回退。

## 工作方式

1. 先阅读 `index.html`、`styles.css`、`app.js`。
2. 每次只完成一个任务或一个可审查的小步骤。
3. 修改前列出将改变的文件。
4. 修改后运行本地静态服务器，并用浏览器检查桌面与移动端。
5. 给出：改动摘要、测试结果、尚未解决的问题。
